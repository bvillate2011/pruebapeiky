Que es responsabilidad unica

Rta Es uno de los principios de SOLID que lo que quiere decir es que una clase solo deberia ser responsable de una cosa, esto para facilitar
que nuestro codigo pueda ser testeable y entendible, y no tener una clase que hace de todo.

Codigo Limpio

Yo considero que un codigo limpio es un codigo que se puede entender y no necesariamente por que se hagan cosas sencillas, sino por que asi 
sean complejas podemos llegar a programarlas de tal manera que otro programador vea mi codigo y lo pueda entender. Ademas un codigo limpio
tambien debe ser facil de testear y que permita hacer cambios sin que se vea afectado el codigo de manera compleja
